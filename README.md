# host-checker

Маленькая CLI-утилита для проверки доступности хостов (ping) из списка.  
Фокус: **простота запуска**, читаемые логи, понятные инструкции.

---

## Зачем

- Быстро понять: **что из списка хостов пингуется, а что — нет**.
- При необходимости сохранить результат в файл для отчёта/аналитики.
- Показать аккуратный репозиторий с прозрачным README.

---

## Требования

- Python **3.10+**
- Системная команда `ping` должна быть в `PATH`  
  - Windows: встроено.  
  - Linux: пакет `iputils-ping` (обычно уже установлен).

Подсказки по платформам
Windows: ping принимает таймаут в миллисекундах (ключ /w).

Linux: ping может требовать RAW-сокеты (иногда нужен sudo).
Удобнее вызывать системный ping через subprocess без привилегий.
Таймаут обычно через -W (секунды).

FAQ
Почему некоторые домены пингуются нестабильно?
CDN, фильтры ICMP, блокировки на стороне провайдера. Пинг — не гарантия доступности сервиса (только сети/хоста).

Можно ли запускать проверки параллельно?
Сейчас — последовательно (как в коде). Параллельность можно добавить через concurrent.futures — см. Roadmap.

Нужен ли requirements.txt?
Нет, проект на стандартной библиотеке. Если добавите тесты/линтеры — тогда да.

Roadmap
 CLI-параметры (argparse):
--hosts hosts.csv · --count 3 · --interval 1.0 · --timeout 1000 · --out log.csv

 CSV-лог с заголовком (timestamp,host,status,rtt_ms) и корректным flush.

 Параллельные проверки (ThreadPool) с ограничением --workers.

 Поле “описание” во входном CSV: host,desc → логировать обе колонки.

 Коды возврата процесса: 0 (всё ок), 1 (есть недоступные), 2 (ошибка ввода).

 GitHub Actions (CI): линтер/тесты по push.
